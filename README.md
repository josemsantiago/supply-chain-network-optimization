# Regional Supply-Chain Network Optimization

Design and cost-optimization of a U.S. distribution network in Python, comparing
**linear programming, integer programming, metaheuristics, and a gradient-based
continuous-location method** on one real problem.

## What this does

Builds a capacitated facility-location problem from **real data** (2020 U.S.
Census population + population-center coordinates for 49 demand zones; a Bureau of
Transportation Statistics freight rate; FMCSA hours-of-service for the delivery
promise), then decides which of 14 candidate distribution centers to open and how
to route every zone's demand at least total cost. The same instance is solved four
ways and stress-tested with a full sensitivity analysis.

**Methods:** LP relaxation (weak vs. strong) · exact MILP (HiGHS branch-and-bound)
· genetic algorithm · simulated annealing · Weiszfeld center-of-gravity.

## Quick start

```bash
pip install -r requirements.txt          # numpy scipy(>=1.9) pandas matplotlib
cd src && python3 run_analysis.py        # ~13 s; regenerates every figure, table, results.json
```

The Census data is cached to `data/` so runs are offline and deterministic
(master seed `20260712`). `scipy.optimize.milp` needs SciPy ≥ 1.9 (HiGHS).

## Headline result

The exact optimum opens **8 distribution centers** (California, Colorado, Florida,
Georgia, Illinois, New Jersey, Texas, Washington) at **\$51.9 M/yr**, found by
branch-and-bound in ~8 ms. A **strong** LP formulation is exactly integral while
the **weak** one understates cost by 49%; both metaheuristics recover the optimum
on the base instance (and trail from under 1% for annealing to 9% for the genetic
algorithm on a harder one where the exact solver still finishes instantly); and the
**gradient-based continuous optimum lands within about 1%** of the discrete network,
rediscovering the same eight regions. The network is
distance-driven, not capacity-driven, and the optimal count climbs from 5 to 12
centers as freight rates double.

## Reproducibility

Every figure, table, and `results.json` value is regenerated by a single
deterministic run of `src/run_analysis.py` under the fixed master seed. The
cached input data lives in `data/`; nothing else is required to reproduce the
results end to end.
